# Feedback and issue reports for Nova

Thank you for your interest in Nova. Nova is open source, but it is not currently accepting external code contributions, documentation patches, or pull requests.

Feedback is welcome. Bug reports, reproducible test cases, compatibility notes, feature requests, design feedback, and usage reports all help improve Nova.

Opening an issue, commenting on an issue, or sharing feedback does not make you a Nova contributor, maintainer, team member, employee, agent, or project representative. Project direction, implementation decisions, release timing, and merge decisions remain with the repository owner.

## Code contributions

Nova is not currently accepting external pull requests or code patches.

Please do not open pull requests unless the repository owner explicitly asks you to do so for a specific change. Unsolicited pull requests may be closed without review.

Please also avoid posting substantial code implementations in issues. Short snippets, logs, configuration examples, and reproduction cases are welcome when they are needed to explain a bug or compatibility issue.

## What is welcome

### Bug reports

Good bug reports are specific and reproducible. Please include:

- What you expected to happen
- What actually happened
- Steps to reproduce the issue
- Your Obsidian version
- Your Nova version
- Your operating system
- The AI provider and model involved, if relevant
- Screenshots, logs, or sample notes when they help explain the issue

Please remove private vault content, API keys, license keys, and other sensitive information before posting.

### Feature requests

Feature requests are welcome as issues. Please describe:

- The problem or workflow you want Nova to improve
- The behavior you expected
- Any constraints that matter, such as privacy, local-only use, mobile support, or provider compatibility

Feature requests may inform Nova's roadmap, but opening a request does not create an obligation to implement it.

### Compatibility reports

Reports about Obsidian versions, themes, platforms, local models, providers, and API-compatible services are useful. Please include enough detail for the behavior to be reproduced.

### Documentation feedback

If documentation is unclear, outdated, or missing important setup details, please open an issue describing the problem. Documentation pull requests are not currently accepted.

## Issue guidelines

- Search existing issues before opening a new one.
- Keep one issue focused on one bug, request, or compatibility topic.
- Use clear titles.
- Include reproduction steps for bugs.
- Be respectful and constructive.
- Do not post secrets, private notes, or sensitive data.
